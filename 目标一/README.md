# 第一目标优化wzy

本文件夹用于存放 multi-agent A* 集群路径规划的第一目标优化结果。

## 文件说明

本项目包含两个 Python 脚本：

- `multi_agent_A_star (1).py`：原始版本，采用按车辆优先级逐个规划路径的方式。
- `multi_agent_A_star_first_objective.py`：第一优化目标版本，采用集中式 multi-agent A*，目标是让集群路径总长度最短。
- `first_objective_result.html`：第一优化目标结果展示文件，用动画方式展示集群中所有车辆的时空路径。

优化后的结果文件是：

```text
multi_agent_A_star_first_objective.py
```

如果需要查看或运行第一目标优化后的算法，请使用该文件。

如果需要查看动画展示结果，请打开：

```text
first_objective_result.html
```

HTML 页面中提供了文件选择下拉框，可以分别查看：

- `multi_agent_A_star (1).py` 的原始规划效果。
- `multi_agent_A_star_first_objective.py` 的第一目标优化效果。

## 第一优化目标

第一优化目标为：

> 集群路径总长度最短。

在 `multi_agent_A_star_first_objective.py` 中，路径长度按车辆实际移动次数计算：

- 车辆从一个网格移动到相邻网格，路径长度加 1。
- 车辆在原地停留，路径长度加 0。
- 集群路径总长度等于所有车辆路径长度之和。

因此，车辆可以在路径上停留避让，但停留不会增加第一优化目标的代价。

## 冲突约束

第一优化目标版本在搜索过程中检查以下冲突：

- 顶点冲突：同一时刻两辆车不能占用同一个网格。
- 边冲突：两辆车不能在同一时间步内相向交换位置。
- 静态障碍：车辆不能进入地图中值为 `1` 的障碍网格。

## 算法说明

`multi_agent_A_star_first_objective.py` 使用联合状态进行 A* 搜索：

- 一个搜索状态包含所有车辆在同一时刻的位置。
- 每次扩展时，同时枚举所有车辆的下一步动作。
- 动作包括上、下、左、右、原地停留。
- `g` 表示当前集群路径总长度。
- `h` 使用所有车辆到各自目标点的曼哈顿距离之和。

因为 `h` 是剩余移动距离的下界，所以可以用于搜索集群路径总长度最短的解。

在当前目录运行：


## 示例结果

当前示例地图为 `10 x 10` 网格，并在第 4 行设置横向障碍：

```python
grid = [[0] * 10 for _ in range(10)]
grid[4][3:7] = [1, 1, 1, 1]
```

车辆起点和终点如下：

```python
agents = [
    {"id": "A", "start": (0, 0), "goal": (9, 9)},
    {"id": "B", "start": (9, 0), "goal": (0, 9)},
    {"id": "C", "start": (4, 0), "goal": (4, 9)},
]
```

运行结果中的集群总路径长度为：

```text
Cluster total path length: 47
```

各车辆路径长度为：

```text
A Length: 18
B Length: 18
C Length: 11
```

所以：

```text
18 + 18 + 11 = 47
```

该结果满足第一优化目标，即在当前示例地图和车辆任务下，使集群路径总长度最短。

## 优化效果说明

需要说明的是，当前示例中原始版本和第一目标优化版本的集群路径总长度都是 `47`：

```text
原始版本总路径长度：47
第一目标优化版本总路径长度：47
```

因此，在这个简单示例中，优化效果没有体现为“总路径长度数值变小”。

原因是当前地图和车辆任务下，三辆车各自绕开静态障碍所需的最短路径长度分别为：

```text
A: 18
B: 18
C: 11
```

三者相加正好是：

```text
18 + 18 + 11 = 47
```

这个值已经是集群路径总长度的下界，所以优化算法无法再把总路径长度降到 `47` 以下。

第一目标优化版本体现的主要改进是：

- 从逐车优先级规划改为联合状态 A* 搜索。
- 搜索目标显式定义为集群路径总长度最短。
- 在搜索过程中检查车辆之间的顶点冲突和边冲突。
- 允许车辆原地停留避让，但停留不增加第一目标的路径长度代价。

如果要更明显地体现“优化前后路径长度变短”，需要构造更复杂的地图或车辆任务，使原始逐车规划产生绕路，而联合状态优化能够找到总路径长度更短的组合路径。

## 后续扩展

第二优化目标是：

> 从第一辆车从起点出发到最后一辆车到达终点，集群总耗时尽可能短。

该目标可以作为第一优化目标之后的次级目标继续扩展，即先保证集群路径总长度最短，再在所有最短路径长度方案中选择总耗时更短的方案。
